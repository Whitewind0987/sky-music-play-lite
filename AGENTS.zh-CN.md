# AGENTS.md 中文版

本文件用于约束 Codex 和其他 AI 助手在本项目中的工作方式。任何修改代码或文件之前，都必须先遵守这里的规则。

## 固定技术栈

SkyMusicPlay Lite 必须使用以下技术栈：

- Tauri v2
- React
- TypeScript
- Rust
- Vite
- Git 和 GitHub
- Windows 优先

除非用户明确改变项目方向，否则不要替换技术栈。

不要改用：

- Electron
- Python
- Vue
- Svelte
- Next.js
- Flutter
- Qt
- C#
- Java
- 纯 Rust GUI

## 开发方式

- 项目必须按小阶段逐步开发。
- 每次只做一件小事，让它可以运行或可以明确验收，解释清楚，然后停止。
- 不要重新设计总阶段计划。
- 不要提前实现未来阶段的功能。
- 不要主动进行大规模重构。
- 不要为了“看起来更完整”而添加额外功能。
- `main` 分支必须尽量保持可运行。
- 每个阶段完成后必须停止，等待用户确认后再继续下一阶段。

## Git 安全规则

- 不要运行破坏性 Git 命令。
- 不要运行 `git reset --hard`。
- 不要运行 `git clean -fd`。
- 不要运行 `git rebase`。
- 不要运行 `git push --force`。
- 除非用户明确允许，否则不要执行提交或推送。
- 默认只建议用户手动执行 Git 命令。

## 依赖安全规则

添加任何依赖之前，必须先向用户说明。

说明内容包括：

- 依赖名称
- 为什么需要它
- 现在是否必须添加
- 是否可以不用依赖实现同样目标
- 它会带来什么风险

早期阶段不要为了视觉美化或开发便利添加依赖。

不要主动添加：

- UI 组件库
- 状态管理库
- 动画库
- 日志库
- 数据库库
- 自动化库

## 面向初学者的解释要求

每次修改后，都必须用适合初学者的方式解释：

- 改了什么
- 为什么这样改
- 改了哪些文件
- 如何运行
- 如何验证
- 这次涉及哪些基础概念
- 哪些东西暂时不要改

不要只给代码，不解释原因。

## 当前阶段规则

只实现当前阶段。

如果某个功能有用，但属于后续阶段，必须明确说明它会推迟，不要提前实现。

如果不确定某个功能是否属于当前阶段，就把它当作未来功能，暂时不要实现。

## 第 0 阶段特别规则

第 0 阶段只允许创建或修改项目规则和说明文件。

第 0 阶段不要创建：

- `src/`
- `src-tauri/`
- `package.json`
- Tauri 配置文件
- React 应用代码
- Rust 代码
- 曲谱解析逻辑
- 播放逻辑
- 键盘模拟逻辑
